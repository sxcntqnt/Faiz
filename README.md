# ⚡ FlamOS — Terminal Control Plane

A self-healing, tmux-native process orchestrator with live dashboard,
workspace resurrection, and fuzzy navigation. Born from `rieng.py` and
`tmuxSmtWdw.sh`. Grown into something else entirely.

---

## Install

```bash
cd FlamOS
chmod +x install.sh
sudo ./install.sh
```

---

## File Structure

```
FlamOS/
├── flam.py                # CLI entry point  →  linked to /usr/local/bin/flam
├── install.sh             # one-time setup
├── config/
│   └── services.yaml      # ← all services declared here
├── core/
│   ├── registry.py        # loads + parses services.yaml
│   ├── tmux.py            # tmux primitives
│   ├── orchestrator.py    # up / down / restart logic
│   ├── watchdog.py        # self-healing daemon
│   ├── resurrection.py    # save / restore workspace
│   ├── dashboard.py       # rich live UI
│   └── router.sh          # fuzzy session navigator
├── state/
│   └── workspace.json     # auto-generated by: flam save
└── logs/
    └── flam.log
```

## Commands

| Command | What it does |
|---|---|
| `flam up [service]` | Start one or all services |
| `flam down [service]` | Stop one or all services |
| `flam restart [service]` | Restart one or all services |
| `flam status` | Service status snapshot |
| `flam dash` | Live dashboard (refreshes every 3s) |
| `flam jump <name>` | Fuzzy-navigate to a tmux session |
| `tm <name>` | Shorthand for `flam jump` |
| `flam save` | Snapshot workspace to JSON |
| `flam restore` | Resurrect workspace from snapshot |
| `flam snapshot` | Print last saved snapshot |
| `flam watch` | Start the self-healing watchdog daemon |
| `flam panic` | Open all critical log panes simultaneously |
| `flam doctor` | Full health check report |
| `flam list` | List all registered services |

## Adding a new service

Edit `config/services.yaml`:

```yaml
services:
  auth:
    session: FlamAuth
    command: "node server.js"
    dir: /sec/root/FLTM/auth
    health_check: "curl -sf http://localhost:4000/health -o /dev/null"
    color: magenta
    critical: true
    tags: [app, backend]
    port: 4000
```

Then:

```bash
flam up auth
flam jump auth
```

No other changes needed. The registry drives everything.

## Watchdog

Runs as a foreground daemon (put it in its own tmux window):

```bash
# In a new tmux window or session:
flam watch
```

Or launch it in the background via tmux:

```bash
tmux new-session -d -s FlamWatchdog "flam watch"
```

The watchdog:
- Checks all service health every N seconds (configured in services.yaml)
- Auto-restarts degraded services with cooldown
- Opens a named alert pane in `FlamWatchdog` session when restart limit is exceeded
- Logs all events to `logs/flam.log`

## Resurrection

```bash
# Snapshot current workspace
flam save

# After reboot — bring everything back
flam restore
```

The snapshot captures every tmux session, window, pane, cwd, and running command.
On restore, registry services use their canonical commands. Non-registry sessions
use best-effort last-known state.

## Panic mode

```bash
flam panic           # opens log panes for all CRITICAL services
flam panic --all     # opens log panes for every service
```

All panes land in the `FlamPanic` tmux session. `flam jump panic` to get there.
